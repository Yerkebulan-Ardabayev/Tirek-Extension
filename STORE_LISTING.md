Помогать продавцу Kaspi.kz видеть реальную маржу по товару (с учётом комиссий

## Подробное описание (detailed description)

```
Margli — расширение для продавцов Kaspi.kz. Один гвоздь: показать реальную
прибыль по товару и поймать тех, кто демпингует вашу карточку. Без Excel, без
отдельного кабинета, без передачи логина и пароля.

ЧТО ДЕЛАЕТ

1. Калькулятор реальной маржи
Считает чистую прибыль по товару с учётом всего, что съедает деньги на Kaspi:
- комиссия Kaspi по новой шкале 2026 и НДС на комиссию;
- налоговый режим РК (упрощёнка, розничный, ОУР) по ставкам 2026;
- себестоимость, доставка до Kaspi, реклама, возвраты;
- Kaspi Red рассрочка и СПП.
Видно не «маржу на бумаге», а сколько реально останется.

2. Анти-демпинг на карточке товара
Прямо на странице kaspi.kz/shop/p/... показывает всех продавцов общей карточки
и кто стоит ниже вас. Бейдж и боковая панель: сколько демперов, на сколько
процентов ниже, минимальная цена.

3. Наблюдение и уведомления
Возьмите товар «под наблюдение» и получайте уведомление, когда появляется новый
демпер. Удобно для тех, у кого десятки SKU.

ПОЧЕМУ ЭТО ТОЧНО

Комиссии Kaspi и налоги РК на 2026 год сверены по официальным источникам
(adilet.zan.kz — Налоговый кодекс, kgd.gov.kz — разъяснения, guide.kaspi.kz —
комиссии). Расчёт обновлён под реформу 2026 (НДС 16%, новая шкала комиссий).

ПРИВАТНОСТЬ

- Логин и пароль Kaspi не запрашиваются. Расширение читает только то, что уже
  открыто у вас на странице товара.
- Цены, SKU, имя магазина и ссылки никуда не отправляются.
- Анонимная статистика использования выключена по умолчанию; включается вручную
  в настройках. Что собирается — описано в настройках и в политике приватности.

ТАРИФ

Бесплатно: калькулятор маржи без ограничений и наблюдение за 3 товарами.
Pro (9900 ₸/мес): безлимит товаров под наблюдением и анти-демпинг по всему
портфелю. Оплата через Kaspi-перевод, активация по коду.

СТАТУС

Альфа-версия для закрытого теста. Пишите о проблемах и пожеланиях в поддержку.
```

## Обоснование разрешений (Permissions justification)

Заполнить в разделе ревью CWS. По каждому разрешению — зачем оно.

| Разрешение | Обоснование |
|---|---|
| `storage` | Локально (chrome.storage.local) хранятся настройки селлера (имя магазина, налоговый режим, категория, порог демпинга), себестоимость по SKU для расчёта маржи, список товаров под наблюдением и статус лицензии. На сервер не передаётся. |
| `alarms` | Один будильник раз в сутки — сброс анонимной статистики использования (только если пользователь сам включил телеметрию). Без неё будильник ничего не отправляет. |
| `notifications` | Уведомление о новом демпере по товарам, которые пользователь добавил под наблюдение. |
| `host_permissions: https://kaspi.kz/shop/p/*` | Контент-скрипт читает данные карточки товара (список продавцов и их цены), которые Kaspi уже отрендерил в браузере, чтобы посчитать демпинг и показать маржу на самой странице. Только страницы товара, никакие другие разделы Kaspi не затрагиваются. |

**Remote code:** не используется (весь код в пакете, никакой загрузки скриптов
с внешних адресов).

**Data usage (заявить в форме):**
- Не продаём и не передаём данные третьим лицам.
- Не используем данные для целей, не связанных с заявленным назначением.
- Не используем данные для оценки кредитоспособности / кредитования.

## Privacy Policy (нужна страница по URL)

Требуется публичный URL политики приватности. Вариант: GitHub Pages.
Минимальное содержание (на основе того, что плагин реально делает):

- Какие данные обрабатываются локально: настройки, себестоимость SKU, watchlist,
  лицензия — в chrome.storage.local на устройстве пользователя.
- Что НЕ собирается и не отправляется: цены, SKU, имя магазина, ссылки карточек,
  IP, User-Agent, история просмотров.
- Опциональная анонимная телеметрия (по умолчанию выключена): install_id (UUID),
  версия плагина, счётчики событий (открыл карточку / добавил в наблюдение /
  открыл калькулятор), страна по заголовкам. Включается только вручную.
- Контакт для связи.

Пример URL: `https://yerkebulan-ardabayev.github.io/Margli-Extension/privacy.html`
