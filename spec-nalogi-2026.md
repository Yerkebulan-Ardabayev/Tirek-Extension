# Spec: корректные налоги по всему Казахстану (упрощёнка по регионам + дефекты аудита 2026)

Статус: РЕАЛИЗОВАНО 2026-06-20 (одобрено владельцем). tsc 0, 376 тестов зелёные, build чист.
Дата: 2026-06-20. Связано: налоговый аудит в этом чате, [[feedback_data_freshness_transparency]], [[feedback_verify_sources_and_links]].

## Зачем

Аудит налогового модуля выявил, что калькулятор применяет жёстко зашитые 4% упрощёнки
(`RATES_2026.uproshenka`) для всех. По ст. 726 НК РК 2026 базовая ставка действительно 4%,
но маслихат может менять её на ±50% (диапазон 2-6%), и фактически почти все регионы её СНИЗИЛИ.
Поэтому для большинства Kaspi-селлеров главная цифра продукта (остаток/прибыль/маржа) считается
с завышенным налогом.

Подтверждено по первоисточникам на 2026 год:
- г. Алматы 3%, г. Астана 3%, г. Шымкент 2%.
- Большинство областей и районов 2-3%. Регионов со ставкой выше 4% не выявлено.
- Решение маслихата принимается до 1 декабря предыдущего года (ст. 726 НК РК).

## Что делаем (цель)

1. Дать селлеру задать СВОЮ ставку упрощёнки (выбор владельца: свободный ввод, не справочник регионов).
2. Пробросить эту ставку во все расчёты (калькулятор маржи, обзор магазина).
3. Заодно починить 4 дефекта аудита: порог ИПН для ИП, подписи баз взносов, соцналог ИП на ОУР, ссылки на статьи.
4. Тест-гард на каждую ставку с номером статьи в названии теста.

## Не-цели (сознательно НЕ делаем)

- Полный справочник решений всех ~200 маслихатов (выбран свободный ввод ставки). Возможно позже.
- Зачёт входного НДС для ОУР (остаётся текущее упрощение: НДС 16% от оборота как расход).
- Режим «самозанятые» (ИПН 0% + соцплатежи 1%): для Kaspi-селлеров нерелевантен.
- Начисление налога на упрощёнке при убытке (остаётся текущее поведение: при profit<=0 показываем 0).

## Поток данных

settings.uproshenkaRatePercent (UI) -> доля -> StoreRowInput.uproshenkaRate / MarginInput.uproshenkaRate -> calculateTax(uproshenkaRate). При отсутствии значения везде дефолт `RATES_2026.uproshenka` (4%), поведение не ломается.

## Поведение по умолчанию и честность

- Дефолт 4% (юридическая база). Подсказка прямо говорит, что в большинстве регионов ставка ниже, и просит уточнить.
- Не имитируем авто-актуальность: даём значение + дату сверки + ссылку на список маслихатов (граница честности из [[feedback_data_freshness_transparency]]).

## Сценарии приёмки

1. Селлер-ИП упрощёнка в Алматы ставит 3%: на товаре 10 000 ₸ налог с оборота = 300 ₸ (был 400 ₸). Остаток/прибыль/маржа пересчитываются.
2. Селлер в Шымкенте ставит 2%: налог вдвое меньше дефолта.
3. Поле не тронуто: ставка 4%, расчёты как раньше (обратная совместимость).
4. Форма «ТОО ОУР»: поле ставки упрощёнки скрыто (нерелевантно), налог = КПН 20% + НДС 16%.
5. Карточка «ИП на ОУР»: ИПН показывает порог 230 000 МРП; присутствует строка соцналога 2 МРП.

## Тесты (гард с номером статьи)

- `calculateTax` упрощёнка с `uproshenkaRate: 0.03` даёт 3% (ст. 726).
- дефолт без параметра остаётся 4% (ст. 726).
- ИП на ОУР: отображаемый порог ИПН = 230 000 МРП (ст. 363 п.4); физлица = 8 500 МРП (ст. 363 п.1).
- базы: ОПВ max 50 МЗП, СО min 1 / max 7 МЗП.
- карточка `ip-osnovnoy` содержит соцналог (ст. 557).
- существующие 364 теста остаются зелёными; обновить те, что проверяли старые подписи баз.
