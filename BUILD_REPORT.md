# Margli Chrome Extension: BUILD REPORT

Версия: 0.1.0-alpha.8 (см. `package.json` и `manifest.json`)

Краткая сводка по текущей сборке. Подробности по функциям и источникам
тарифов: `README.md`.

## 1. Что собирается

- Manifest V3, целевой Chrome 116+.
- Bundler: esbuild через `scripts/build.mjs` (без Vite и CRX-плагина),
  он же генерирует PNG-иконки 16/48/128.
- Точки входа: background service worker, контент-скрипт страницы товара,
  popup на React 19 (TypeScript strict).

### Что инжектится на страницы

В alpha.8 поставляется только контент-скрипт страницы товара
`kaspi.kz/shop/p/*` (бейдж реальной маржи, подсветка демперов, боковая
панель со списком продавцов, watchlist, PDF-досье).

Парсер кабинета селлера `/mc/products` и `/mc/orders`
(`src/content/mc-products.ts`, `src/lib/kaspi-mc-parser.ts`) **отключён и
не поставляется**: его DOM-селекторы ещё не проверены на реальном
кабинете. Код остаётся в репозитории как заготовка на будущее.

## 3. Тесты

Vitest, **все юнит-тесты проходят** (`pnpm test` / `npm test`).

Покрыто ядро (чистые функции): расчёт маржи, тарифы Kaspi, налоги РК,
парсинг цен страницы товара, обёртка `chrome.storage`, regex-парсер HTML
в service worker. Раскладку по файлам см. в `src/tests/`. Тарифы и налоги
проверяются строгими guard-тестами на конкретные значения 2026 года и на
запрещённые домены источников.

## 4. Сборка и упаковка

```bash
# в папке margli-extension/
pnpm install        # установить зависимости
pnpm test           # vitest, все тесты зелёные
pnpm typecheck      # tsc --noEmit, 0 ошибок
pnpm build          # esbuild → dist/
pnpm package        # build + zip → releases/margli-extension-v<version>.zip
```

(команды доступны и через `npm run <script>`.)

Две конфигурации сборки:

- **Dev-билд** (`pnpm build` / `npm run build`): отлаживаемый, с
  sourcemap, без минификации. Именно его грузят через Load Unpacked из
  `dist/` при разработке.
- **Prod-пакет** (`pnpm package` / `npm run package`): минифицированный,
  без sourcemap, ужатый zip для раздачи тестерам.

### Загрузка в Chrome (dev)

1. Открыть `chrome://extensions`.
2. Включить «Режим разработчика» (тумблер справа вверху).
3. «Загрузить распакованное» (Load Unpacked).
4. Выбрать папку `dist/`
   (`C:\Users\yerke\Desktop\Margli\margli-extension\dist\`).

После изменений в `src/` пересобрать (`pnpm build`) и нажать «Обновить»
у Margli в `chrome://extensions`.

## 5. Известные ограничения

- Парсер кабинета `/mc/*` отключён (см. п. 1): селекторы не верифицированы
  на реальном кабинете.
- DOM-селекторы страницы товара построены на списках fallback-вариантов;
  при смене вёрстки Kaspi их нужно дополнять (см. раздел «При изменении
  DOM Kaspi» в `README.md`).
- PDF-досье использует встроенный шрифт jsPDF (helvetica): кириллица
  читается, но без идеального хинтинга.
- Watchlist обновляется пассивно: данные по отслеживаемому SKU приходят,
  когда вы сами открываете его карточку. Фоновой перепроверки скрытым окном
  в этой сборке нет (убрана перед подачей в Web Store). Единственный фоновый
  таймер `chrome.alarms` это раз в сутки flush анонимной телеметрии. Кнопка
  «Игнорировать» в уведомлении завязана на жизненный цикл MV3 service worker.
- Все строки UI захардкожены на русском, `_locales/` нет.
- Категории Kaspi укрупнены до ~20 групп вместо полной таблицы 2026 года
  (полная отдаётся только в партнёрском PDF).
