# Margli Chrome Extension — BUILD REPORT

Дата сборки: 2026-05-05
Версия: 0.1.0 (MVP)

## 1. Что сделано — по 5 пунктам функциональности

### ★ 1. Overlay на странице товара (`kaspi.kz/shop/p/*`)

- Content script `src/content/shop-page.ts` инжектится при загрузке страницы
- Парсер `src/lib/kaspi-shop-parser.ts` ждёт через `MutationObserver` пока
  Kaspi отрендерит блок продавцов (max 15 сек)
- Бейдж в правом нижнем углу: «🛡 Margli: N демперов, −X%» / «✅ Конкурентов нет»
- Боковая панель (drawer) с таблицей всех продавцов, отсортированной по цене
- Кнопки ⭐ «Следить» и 📄 «Досье жалобы»
- **Изоляция CSS через Shadow DOM** — стили Kaspi не могут протечь
- Парсер использует **списки fallback-селекторов** (5+ вариантов на каждое поле)
  и JSON-LD как запасной источник
- Подробное логирование в console какой селектор сработал

### ★ 2. Бейджи в кабинете селлера (`/mc/products`, `/mc/orders`)

- Content script `src/content/mc-products.ts`
- Парсер `src/lib/kaspi-mc-parser.ts` (селекторы — гипотетические, помечены TODO)
- К каждой строке таблицы добавляется бейдж:
  - 📊 Маржа в % (если задана себестоимость) — цветовая индикация ok/warn/danger
  - ⚠ Демпер −X% (если SKU в watchlist и есть демперы)
- На странице заказов — бейдж «чистая прибыль с заказа»
- Стили инжектятся через `<style>` тег с уникальным ID

### ★ 3. Калькулятор реальной маржи

- React-компонент `src/popup/MarginCalculator.tsx`
- Чистая функция `src/lib/margin-calc.ts` — `calculateMargin(input)` → `MarginResult`
- 9 параметров ввода: цена / категория / закупка / доставка / реклама / возвраты /
  налоговый режим / Kaspi Red / СПП
- Полная разбивка с группами: kaspi-fees / ops / cost / tax / result
- Реактивный пересчёт при любом изменении формы

### ★ 4. Popup-дашборд

- 4 вкладки в `src/popup/App.tsx`: Сегодня / Наблюдение / Калькулятор / Настройки
- **Сегодня** — KPI: новые демперы (24ч), под наблюдением, всего демперов, средняя маржа
- **Наблюдение** — список SKU с inline-формой себестоимости (закупка/доставка/реклама/возвраты/категория)
- **Настройки** — myShopId, налоговый режим (с описанием и ссылкой на источник),
  категория по умолчанию, СПП toggle, Kaspi Red toggle, alerts on/off, порог демпинга
- Дизайн повторяет тёмную тему margli-preview (brand-300 #a78bfa accent)

### ★ 5. Background worker + notifications + PDF-досье

- `src/background/worker.ts` — MV3 service worker
- `chrome.alarms.create("margli:recheck", { periodInMinutes: 30 })`
- При срабатывании alarm — fetch'ит каждый watched URL, парсит конкурентов
  через regex (`src/background/fetch-helper.ts` — без DOMParser, не везде доступен в SW)
- Сравнивает с `lastSeen`, находит **новых** демперов (не было раньше или цена снизилась)
- `chrome.notifications.create()` с двумя кнопками: «Открыть» / «Игнорировать»
- Клик «Открыть» → открывает страницу товара в новой вкладке
- Клик «Игнорировать» → добавляет shopId в blacklist для конкретного SKU
- PDF-досье в `src/lib/pdf-dossier.ts` через **jsPDF**:
  - A4, шапка с заголовком и датой
  - Раздел «Товар» (название/SKU/URL)
  - Раздел «Заявитель» (магазин/цена)
  - Таблица демперов с дельтами и кол-вом отзывов
  - Раздел подписи и описания нарушения

## 5. Известные TODO после реального тестирования в Chrome

1. **Селекторы кабинета селлера `/mc/products` и `/mc/orders`** —
   `src/lib/kaspi-mc-parser.ts` использует гипотетические имена классов.
   Юзер должен:
   - Открыть кабинет → DevTools → найти строку товара
   - Скопировать `class` атрибут `<tr>`
   - Добавить в начало `PRODUCT_ROW_SELECTORS` (или `ORDER_ROW_SELECTORS`)
   - Аналогично для cell-селекторов внутри строки

2. **PDF-досье — кириллица в helvetica** — jsPDF встроенный шрифт рендерит
   кириллицу, но без идеального хинтинга. Можно подключить Inter через
   `doc.addFont()` для лучшего вида. Сейчас работает, читается.

3. **Notification «Игнорировать» — теряется при рестарте Chrome** —
   ограничение MV3 service worker'а. После перезапуска браузера старый
   notificationId забывается. Текущая логика: если notification ещё в
   shadebar и юзер кликает «Игнорировать» — работает корректно.

4. **Recheck 30 мин — без backoff** — при 100+ watched SKU и медленной
   сети Kaspi'я проход может затянуться. В v0.2 добавить параллелизм
   (Promise.all с лимитом) и exponential backoff на ошибки.

5. **DOM Kaspi на странице товара** — точные селекторы в
   `kaspi-shop-parser.ts` (например `.sellers-table__row`) основаны на
   типичных паттернах Kaspi'а, но WebFetch не отдаёт JS-rendered HTML.
   При первом запуске на реальном товаре console покажет какой селектор
   сработал — если ни один, юзер увидит «no competitor rows found» в логе
   и пришлёт мне DevTools-снимок.

6. **i18n** — все строки UI хардкодом на русском. Целевая аудитория —
   казахстанские селлеры, многие используют русский. При желании
   локализовать на казахский — добавить `_locales/` папку и `chrome.i18n.getMessage()`.

7. **Категории Kaspi** — 20 укрупнённых vs 237 фактических. Если селлер
   считает что его подкатегория не подходит — он выберет «Прочее» (13.5%)
   и увидит маржу с пессимистичной оценкой.

### Загрузка в Chrome

1. `chrome://extensions` в адресной строке
2. Включить **«Режим разработчика»** (тумблер в правом верхнем углу)
3. **«Загрузить распакованное»** (Load Unpacked)
4. Выбрать папку `dist/` (полный путь:
   `C:\Users\yerke\Desktop\Margli\margli-extension\dist\`)
5. Иконка Margli (фиолетовый градиент с буквой M) появится в toolbar Chrome
6. Открыть любой товар на kaspi.kz/shop/p/* — в правом нижнем углу появится бейдж

### Обновление после изменения кода

После любого `Edit` в src/:
```bash
pnpm build
```
Затем в `chrome://extensions` нажать «Обновить» (🔄) у Margli.
